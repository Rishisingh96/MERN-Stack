# 📘 1. What is JavaScript?

## Definition

JavaScript is a **high-level, interpreted, lightweight, dynamically typed programming language** mainly used to make web pages interactive and dynamic.

Initially JavaScript was created only for browsers, but today it is used for:

* Frontend Development
* Backend Development
* Mobile Apps
* Desktop Applications
* Game Development
* AI/ML Applications

JavaScript is one of the most popular programming languages in the world.

---

## Simple Explanation

HTML webpage ka structure banata hai.
CSS webpage ko style karta hai.
JavaScript webpage ko **dynamic aur interactive** banata hai.

Example:

* Button click hone par action lena
* Form validation
* API se data fetch karna
* Live chat banana
* Animation create karna

Without JavaScript, websites mostly static hoti.

---

## Example Without JavaScript

```html
<button>Click Me</button>
```

Button page par dikhega but click karne par kuch nahi hoga.

---

## Example With JavaScript

```html
<button onclick="showMessage()">Click Me</button>

<script>
function showMessage() {
    alert("Hello User!");
}
</script>
```

Ab button click karne par popup aayega.

---

## History of JavaScript

JavaScript ko **Brendan Eich** ne 1995 me sirf **10 days** me banaya tha at **[Netscape](https://www.netscape.com?utm_source=chatgpt.com)**.

Initial names:

* Mocha
* LiveScript
* JavaScript

Important:
JavaScript ka **Java se direct relation nahi hai**.

Sirf marketing ke liye naam JavaScript rakha gaya.

## Why JavaScript is Popular

Reasons:

* Full-stack development possible

Example:

* automate tasks
* control another software
* dynamic behavior add karna

* directly interpreted hoti thi

Languages like:

* C
* C++
* Java

# 📘 5. What is the difference between JavaScript and ECMAScript?

Ye interview me bahut common question hai.

Most beginners JavaScript aur ECMAScript ko same samajhte hain, but technically dono alag hain.

---

# Simple Definition

### JavaScript

JavaScript ek programming language hai jo developers use karte hain.

Example:

```javascript
let name = "Rishi";
console.log(name);
```

---

### ECMAScript

ECMAScript ek standard/specification hai jo define karta hai ki JavaScript language ka behavior kaisa hona chahiye.

It defines:

* syntax rules
* keywords
* functions
* language standards

---

## Easy Analogy

Think like this:

* ECMAScript = Rule Book
* JavaScript = Actual Implementation

Example:

Suppose cricket ke rules ICC banati hai.

* ICC rules = ECMAScript
* India/Australia cricket teams = JavaScript engines implementing rules

Same way:

* **Ecma International** defines standards
* Browsers and engines implement them

Official:
[ECMA International](https://ecma-international.org?utm_source=chatgpt.com)

---

# Why ECMAScript Was Needed?

Early days me problem thi:
Different browsers JavaScript ko different ways me interpret karte the.

Example:

* Browser A → code runs
* Browser B → same code fails

Standardization ke liye ECMAScript introduce hua.

---

# Example

ECMAScript says:
`let`, `const`, arrow functions supported hone chahiye.

Example:

```javascript
const greet = () => {
   console.log("Hello");
};
```

Browser ka JavaScript engine ensure karega ki ye standard ke according run kare.

---

# Key Differences

| Feature | JavaScript                 | ECMAScript                        |
| ------- | -------------------------- | --------------------------------- |
| Type    | Programming Language       | Specification/Standard            |
| Purpose | Used to build applications | Defines language rules            |
| Used By | Developers                 | Browser vendors & engine creators |
| Example | Writing code               | Defining syntax                   |

# Real-World Implementations

* Chrome V8

* Safari JavaScriptCore

Initially JavaScript purely interpreted language thi.

* execute hota tha immediately

Example:

```javascript

```

Interpreter directly execute karta.

# What is an Interpreter?

Interpreter:

* code read karta
* line by line execute karta
* separate executable file nahi banata

Example:

```javascript
let x = 10;
console.log(x);
```

Execution:

1. Read line 1
2. Execute
3. Read line 2
4. Execute

---

# What is a Compiler?

Compiler:

* entire code ko machine code me convert karta
* phir machine code run hota

Example languages:

* C
* C++
* Java (bytecode + JVM)

---

# Modern JavaScript Reality

Modern JavaScript engines like:

* Chrome → V8
* Firefox → SpiderMonkey
* Safari → JavaScriptCore

Use:

* Interpreter
* Compiler
* JIT (Just-In-Time Compilation)

Together.

---

Example:

```javascript
function add(a, b) {

add(10, 20);
```

Execution flow:

### Step 1: Parsing

Engine code parse karta hai.

AST = Abstract Syntax Tree

Code structure tree me convert hota hai.

---

### Step 3: Interpreter

Initially code interpret hota hai.

Quick execution start hota hai.

---

### Step 4: JIT Compiler

Frequently used code optimize hota hai.

Example:

```javascript
for(let i=0; i<100000; i++){
   add(i, i);

```

Agar function repeatedly run ho raha hai, engine optimize karega.

Compiled machine code banega.

JavaScript dono ka best combination use karta hai.


Number → String me convert hua.


Output:

```javascript
3
```

Why?
String → Number convert hua.

---

# Example 3

```javascript

```

Output:

```javascript
2
```

Because:

* true = 1

| ---------- | ------ |

JavaScript tries to be flexible.

# 21. What is a Variable in JavaScript?

A **variable** is a container used to store data values in memory so that they can be used and modified later in a program.

In JavaScript, variables allow us to store:

* Numbers
* Strings
* Arrays
* Objects
* Functions
* Boolean values
* Null / Undefined

---

## Why do we need variables?

Imagine writing a program without variables.

```js
console.log("Rishi");
console.log("Rishi");
console.log("Rishi");
```

This is repetitive.

Using a variable:

```js
let name = "Rishi";

console.log(name);
console.log(name);
console.log(name);
```

Much cleaner and reusable.

---

## Syntax of Variable Declaration

JavaScript provides 3 ways to declare variables:

```js
var name = "Rishi";
let age = 24;
const country = "India";
```

General syntax:

```js
keyword variableName = value;
```

Example:

```js
let city = "Indore";
```

Here:

* `let` → keyword
* `city` → variable name
* `"Indore"` → value

---

## Variable Declaration vs Initialization

### Declaration

Creating variable name.

```js
let age;
```

---

### Initialization

Assigning value.

```js

```

# 22. What is the Difference Between var, let, and const?

This is one of the most important JavaScript interview questions.

JavaScript has 3 ways to declare variables:

* `var` → old way (before ES6)
* `let` → modern way
* `const` → constant values

---

# 1. var

Introduced in old JavaScript.

Example:

```js
var name = "Rishi";
```

Properties:

* Function scoped
* Can be redeclared
* Can be updated
* Hoisted with undefined

Example:

```js
var age = 20;
age = 25; // allowed
```

Redeclare:

```js
var age = 20;
var age = 25; // allowed
```

---

# 2. let

Introduced in ES6.

Example:

```js
let age = 24;
```

Properties:

* Block scoped
* Cannot redeclare
* Can update
* Hoisted but in Temporal Dead Zone

Update:

```js
let age = 24;
age = 25;
```

Allowed.

Redeclare:

```js
let age = 24;
let age = 30; // Error
```

Not allowed.

---

# 3. const

Introduced in ES6.

Example:

```js
const PI = 3.14;
```

Properties:

* Block scoped
* Cannot redeclare
* Cannot reassign
* Must initialize during declaration

Example:

```js
const PI = 3.14;
PI = 4; // Error
```

---

## Comparison Table

| Feature                | var      | let   | const |
| ---------------------- | -------- | ----- | ----- |
| Scope                  | Function | Block | Block |
| Redeclare              | Yes      | No    | No    |
| Reassign               | Yes      | Yes   | No    |
| Hoisting               | Yes      | Yes   | Yes   |
| Initial Value Required | No       | No    | Yes   |

---

## Scope Example

```js
if (true) {
  var a = 10;
  let b = 20;
  const c = 30;
}

console.log(a); // 10
console.log(b); // Error
console.log(c); // Error
```

Why?

Because:

* `var` ignores block scope
* `let` and `const` respect block scope

## Const with Objects

YES.

Works.

Because `const` prevents reassignment, not mutation.

Wrong:

```js

```

Error.

## Which One Should You Use?

Best practice:

* Use `let` if value changes
* Avoid `var`


`var` is function-scoped and allows redeclaration.

# 29. What is Lexical Scope in JavaScript?

Lexical Scope means that the accessibility of variables is determined by **where variables and functions are written in the code (their physical location)**.

In simple words:

A function can access variables from:

* Its own scope
* Outer (parent) scope
* Global scope

This behavior is called **Lexical Scope**.

---

# Meaning of “Lexical”

Lexical means:

> Related to the structure/location of code.

JavaScript decides scope during **code writing time**, not runtime.

---

# Basic Example

```js
let globalVar = "I am global";

function outer() {
  let outerVar = "I am outer";

  function inner() {
    let innerVar = "I am inner";

    console.log(innerVar);
    console.log(outerVar);
    console.log(globalVar);
  }

  inner();
}

outer();
```

Output:

```js
I am inner
I am outer
I am global
```

---

# How It Works

When `inner()` executes:

JavaScript searches variables in this order:

1. Inner scope
2. Outer scope
3. Global scope

This is called the **Scope Chain**.

---

# Scope Chain Visualization

```js
Global Scope
   ↓
Outer Function Scope
   ↓
Inner Function Scope
```

Search starts from bottom and moves upward.

---

# Example of Scope Chain

```js
let a = 10;

function first() {
  let b = 20;

  function second() {
    let c = 30;

    console.log(a);
    console.log(b);
    console.log(c);
  }

  second();
}

first();
```

Output:

```js
10
20
30
```

---

# Reverse Access is NOT Possible

Outer function cannot access inner function variables.

Example:

```js
function outer() {
  function inner() {
    let x = 10;
  }

  console.log(x);
}

outer();
```

Output:

```js
ReferenceError
```

Why?

Because scope only works upward, not downward.

---

# Real-Life Example

Think of lexical scope like family hierarchy:

* Child can access parent resources
* Parent cannot access child’s private stuff

Example:

* Inner function = child
* Outer function = parent

---

# Lexical Scope + Closures

Lexical scope is the foundation of **closures**.

Example:

```js
function outer() {
  let counter = 0;

  return function inner() {
    counter++;
    console.log(counter);
  };
}

const increment = outer();

increment();
increment();
increment();
```

Output:

```js
1
2
3
```

Why?

Because inner function remembers outer function variables due to lexical scope.

---

# Interview Answer (Short)

```js

```

Inner `x` shadows outer `x`.

Nearest match wins.

# Definition

A closure is created when an inner function remembers and accesses variables from its outer function even after the outer function has finished execution.

In simple words:

> Closure = Function + Lexical Environment

---


* Local variables are destroyed.

Because inner function still references it.

That preserved environment is closure.

```js id="wnlnwd"

```

Returns inner function.

```js id="mjlwm2"

```

### Step 3

Even though `outer()` is done, `inner()` remembers `message`.

```js id="gdbn2n"
}
```

```js id="mq7w7r"
Outer
Global
```

# How Lookup Works

For:

```js id="m8h1yh"

```

JavaScript checks:

Step 1:

* Inner scope

Not found.

Step 2:

* Outer scope

Not found.

Step 3:

* Global scope

Found.

```js id="dxjvvh"
Inner Scope
   ↓

   ↓
```

# 40. How Does Garbage Collection Work?

Garbage Collection (GC) is JavaScript’s automatic memory management system.

It removes unused memory automatically.

This prevents memory from filling up unnecessarily.

---

# Why Garbage Collection?

Suppose:

```js id="g7p9gy"
let user = {
  name: "Rishi"
};
```

Memory allocated.

Later:

```js id="3kk82q"
user = null;
```

Now object becomes unreachable.

No code can access it.

JavaScript identifies this unused memory and removes it.

---

# Definition

Garbage collection is the process of automatically freeing memory occupied by objects that are no longer reachable.

---

# Reachability

Important concept.

An object is kept in memory if it is reachable.

Reachable means accessible from:

* Global variables
* Current function
* Other reachable objects

Example:

```js id="qglmyx"
let obj = {
  name: "Rishi"
};
```

Reachable → memory retained.

---

If:

```js id="8h7w2l"
obj = null;
```

Unreachable → eligible for garbage collection.

---

# Mark-and-Sweep Algorithm

Modern JavaScript engines use **Mark-and-Sweep**.

Steps:

---

## Step 1: Mark

GC marks all reachable objects.

Example roots:

* Global variables
* Current stack variables

---

## Step 2: Sweep

Unreachable objects are removed from memory.

---

# Example

```js id="j8w7ay"
let user = {
  address: {
    city: "Indore"
  }
};
```

Both reachable.

If:

```js id="9j0vml"
user = null;
```

Now:

* user object unreachable
* address object unreachable

GC removes both.

---

# Memory Leak

When unused memory is not released.

Common causes:

* Global variables
* Unremoved event listeners
* Closures holding references
* Timers

Example:

```js id="4gggqp"

```

If references remain, memory stays occupied.

---

# Example Leak

```js id="k1r4h7"

```

If cache grows forever → memory leak.

---

# Interview Answer (Short)

Garbage collection is JavaScript’s automatic memory cleanup process. It uses algorithms like Mark-and-Sweep to remove unreachable objects from memory.

# 45. Difference Between Arrow Functions and Regular Functions

This is a VERY common interview question.

---

# 1. Syntax Difference

Regular Function:

```javascript
function add(a, b) {
    return a + b;
}
```

Arrow Function:

```javascript
const add = (a, b) => a + b;
```

Arrow is shorter.

---

# 2. `this` Binding (MOST IMPORTANT)

Regular function has its own `this`.

Arrow function does NOT have its own `this`.

It inherits `this` from parent scope.

---

## Regular Function Example

```javascript
const user = {
    name: "Rishi",
    greet: function() {
        console.log(this.name);
    }
};

user.greet();
```

Output:

```javascript
Rishi
```

Here `this = user`

---

## Arrow Function Example

```javascript
const user = {
    name: "Rishi",
    greet: () => {
        console.log(this.name);
    }
};

user.greet();
```

Output:

```javascript
undefined
```

Why?

Arrow function doesn’t create its own `this`.

---

# Best Example for Interview

```javascript
function Person() {
    this.age = 0;

    setInterval(() => {
        this.age++;
        console.log(this.age);
    }, 1000);
}

new Person();
```

Arrow function here captures parent `this`.

Very useful.

---

# 3. Arguments Object

Regular function has `arguments`.

```javascript
function test() {
    console.log(arguments);
}

test(1, 2, 3);
```

Output:

```javascript
[1, 2, 3]
```

---

Arrow function does NOT have arguments.

```javascript
const test = () => {
    console.log(arguments);
};
```

Error.

Instead use rest parameter:

```javascript
const test = (...args) => {
    console.log(args);
};
```

---

# 4. Constructor Usage

Regular function can be constructor.

```javascript
function Person(name) {
    this.name = name;
}

const p = new Person("Rishi");
```

Works.

---

Arrow function cannot be constructor.

```javascript
const Person = (name) => {
    this.name = name;
};

new Person("Rishi");
```

Output:

```javascript
TypeError
```

---

# 5. Hoisting

Regular function declaration is hoisted.

```javascript
hello();

function hello() {
    console.log("Hello");
}
```

Works.

Arrow functions are not fully hoisted.

```javascript
hello();

const hello = () => {
    console.log("Hello");
}
```

Error.

---

# 6. Suitable Use Cases

### Use Regular Functions:

* Object methods
* Constructors
* Functions requiring dynamic `this`

---

### Use Arrow Functions:

* Callbacks
* Short functions
* Functional programming
* React components

Example:

```javascript
numbers.map(num => num * 2);
```

# Final Comparison Table

| Feature     | Regular Function  | Arrow Function |
| ----------- | ----------------- | -------------- |
| Syntax      | Longer            | Short          |
| this        | Own               | Lexical        |
| arguments   | Available         | Not available  |
| Constructor | Yes               | No             |
| Hoisting    | Yes (declaration) | No             |

**Function Declaration:** Named function defined using `function`, fully hoisted.
**Function Expression:** Function stored inside variable, not fully hoisted.

# 49. What is Recursion?

Recursion means a function calling itself until a stopping condition is met.

Simple meaning:

Function repeatedly calls itself to solve smaller versions of same problem.

```javascript

    if(condition) {
        return;
    }

}
```

Important parts:

* Base condition
* Recursive call

Without base condition → infinite loop / stack overflow.

---

# Example 1: Count Down

```javascript
function countDown(n) {
    if (n === 0) {
        console.log("Done");
        return;
    }

    console.log(n);
    countDown(n - 1);
}

countDown(5);
```

Output:

```javascript

4
3
2
1
Done
```

Formula:

```javascript
5! = 5 × 4 × 3 × 2 × 1
```

---

Code:

```javascript
function factorial(n) {
    if (n === 1) {
        return 1;
    }

    return n * factorial(n - 1);
}

console.log(factorial(5));
```

Output:

```javascript
120
```

---

# Step-by-Step

```javascript
factorial(5)
= 5 * factorial(4)
= 5 * 4 * factorial(3)
= 5 * 4 * 3 * factorial(2)
= 5 * 4 * 3 * 2 * factorial(1)
= 120
```

---

# Example 3: Fibonacci


```javascript
8
```

# 50. What is Function Currying?

Currying means transforming a function with multiple arguments into a sequence of functions, each taking one argument.

---

# Curried Version

```javascript
function add(a) {
    return function(b) {

            return a + b + c;
        };
    };

console.log(add(1)(2)(3));
```

Output:

```javascript

```

---

```javascript

```

Breakdown:

```javascript
add(1) → returns function
(2)     → returns function
(3)     → returns result
```

---

# Arrow Function Version

```javascript
const add = a => b => c => a + b + c;

```

* Code reusability
* Function composition
* Partial application
* Cleaner functional code

```javascript

    return a * b;
}
```

---

With currying:


# 51. What is Partial Application?

Partial Application is a technique where we fix (pre-fill) some arguments of a function and return a new function that accepts the remaining arguments.

Simple meaning:

Instead of passing all arguments at once, we pass some now and some later.

---

# Example Without Partial Application

```javascript
function multiply(a, b) {
    return a * b;
}

console.log(multiply(2, 5));
```

Output:

```javascript
10
```

---

# Using Partial Application

Suppose we always multiply by 2.

```javascript
function multiply(a, b) {
    return a * b;
}

function double(num) {
    return multiply(2, num);
}

console.log(double(5));
```

Output:

```javascript
10
```

Here:

* First argument is fixed = 2
* Remaining argument is passed later

This is partial application.

---

# Generic Example

```javascript
function greet(greeting, name) {
    return `${greeting}, ${name}`;
}
```

Without partial application:

```javascript
console.log(greet("Hello", "Rishi"));
```

Output:

```javascript
Hello, Rishi
```

---

Now fix greeting:

```javascript
function partialGreet(greeting) {
    return function(name) {
        return greet(greeting, name);
    };
}

const sayHello = partialGreet("Hello");

console.log(sayHello("Rishi"));
console.log(sayHello("Aman"));
```

Output:

```javascript
Hello, Rishi
Hello, Aman
```

---

# Using bind()

JavaScript’s `bind()` supports partial application.

```javascript
function multiply(a, b) {
    return a * b;
}

const double = multiply.bind(null, 2);

console.log(double(5));
```

Output:

```javascript
10
```

Fix one value.

### Currying

One argument per function.

A generator function is a special function that can pause execution and resume later.

Introduced in ES6.

Defined using:

```javascript
function*
```

Generators use:

* `yield`
* `next()`

---

# Syntax

```javascript
function* generatorFunction() {
}
```

---

# Basic Example

```javascript

    yield 1;
    yield 2;
    yield 3;
}
```

Create generator:

```javascript

```

---

# Using next()

```javascript
console.log(gen.next());
console.log(gen.next());
console.log(gen.next());
console.log(gen.next());
```

Output:

```javascript
{ value: 1, done: false }
{ value: 2, done: false }
{ value: 3, done: false }
{ value: undefined, done: true }
```

---

# How It Works

When function reaches:

```javascript
yield
```

execution pauses.

Calling:

```javascript
next()
```

resumes execution.

---

# Example

```javascript
function* test() {
    console.log("Start");
    yield 1;

    console.log("Middle");
    yield 2;

    console.log("End");
}
```

```javascript

gen.next();
gen.next();
```

Output flow:

```javascript
Start
Middle
End
```

```javascript
    while (i <= 3) {
```

# 62. What are Async Functions?

Async functions are special functions that simplify asynchronous programming.

They were introduced in ES2017.

Defined using:

```javascript
async
```

Inside async functions, we can use:

```javascript
await
```

---

# Why Needed?

Before async/await:

* callbacks
* promises
* callback hell

Async functions make async code look like synchronous code.

---

# Syntax

```javascript
async function myFunction() {
}
```

---

# Example

```javascript
async function greet() {
    return "Hello";
}
```

---

Calling:

```javascript
console.log(greet());
```

Output:

```javascript
Promise { "Hello" }
```

Important:

Async function always returns a Promise.

---

# Example with await

```javascript
function fetchData() {
    return new Promise(resolve => {
        setTimeout(() => {
            resolve("Data received");
        }, 2000);
    });
}
```

---

Using async/await:

```javascript
async function getData() {
    const result = await fetchData();
    console.log(result);
}

getData();
```

Output after 2 sec:

```javascript
Data received
```

---

# Without Async/Await

Using promises:

```javascript
fetchData().then(result => {
    console.log(result);
});
```

---

# With Async/Await

Cleaner:

```javascript
const result = await fetchData();
```

---

# Error Handling

Use try-catch.

```javascript
async function getData() {
    try {
        const result = await fetchData();
        console.log(result);
    } catch (error) {
        console.log(error);
    }
}
```

---

# Real-world Example

API calls.

```javascript
async function getUsers() {
    const res = await fetch("/users");
    const data = await res.json();
    console.log(data);
}
```

---

# Benefits

* Cleaner syntax
* Better readability
* Easier debugging
* Avoid callback hell

# 63. What is Function Borrowing?

Function borrowing means borrowing a method from one object and using it with another object.

Usually done using:

* `call()`
* `apply()`
* `bind()`

---

# Why Needed?

Sometimes one object has a useful method, and another object wants to use it without redefining.

---

# Example

```javascript
const person1 = {
    name: "Rishi",
    greet: function(city) {
        console.log(`Hello, I am ${this.name} from ${city}`);
    }
};
```

---

Another object:

```javascript
const person2 = {
    name: "Aman"
};
```

Borrow method:

```javascript
person1.greet.call(person2, "Indore");
```

Output:

```javascript
Hello, I am Aman from Indore
```

---

# Explanation

Method belongs to:

* person1

But executed for:

* person2

That is function borrowing.

---

# Using apply()

```javascript
person1.greet.apply(person2, ["Bhopal"]);
```

---

# Using bind()

```javascript
const borrowed = person1.greet.bind(person2);

borrowed("Delhi");
```

---

# Real-world Example

Useful when:

* reusing methods
* avoiding duplicate code
* working with shared utilities

---

# Interview One-line

Function borrowing means using methods of one object with another object using call, apply, or bind.

* Cleaner code

* Encapsulation

# 65. What are Constructor Functions?

Constructor functions are regular functions used to create multiple objects using `new`.

Before ES6 classes, constructor functions were commonly used for object creation.

---

# Syntax

By convention:

* Constructor names start with capital letter.

```javascript
function Person(name, age) {
}
```

---

# Example

```javascript
function Person(name, age) {
    this.name = name;
    this.age = age;

    this.greet = function() {
        console.log(`Hello, I am ${this.name}`);
    };
}
```

Create object:

```javascript
const user1 = new Person("Rishi", 22);
const user2 = new Person("Aman", 24);
```

---

# Usage

```javascript
user1.greet();
user2.greet();
```

Output:

```javascript
Hello, I am Rishi
Hello, I am Aman
```

---

# What Happens with `new`?

When we write:

```javascript
new Person("Rishi", 22)
```

JavaScript does:

### 1. Creates empty object

```javascript
{}
```

---

### 2. Sets `this` to new object

---

### 3. Executes function

```javascript
this.name = name;
```

---

### 4. Returns object automatically

---

# Constructor Example

```javascript
function Car(brand) {
    this.brand = brand;
}
```

Create:

```javascript
const car1 = new Car("BMW");
const car2 = new Car("Audi");
```

---

# Problem

Methods get duplicated.

```javascript
this.greet = function() {}
```

Every object gets separate copy.

Solution:
Use prototype.

```javascript
function Person(name) {
    this.name = name;
}

Person.prototype.greet = function() {
    console.log(this.name);
};
```

More memory efficient.

---

# Constructor Function vs Class

ES6 class syntax is cleaner.

Constructor:

```javascript
function Person(name) {
    this.name = name;
}
```

Class:

```javascript
class Person {
    constructor(name) {
        this.name = name;
    }
}
```

Both work similarly behind the scenes.

---

# Factory vs Constructor

| Feature      | Factory Function | Constructor Function |
| ------------ | ---------------- | -------------------- |
| new keyword  | No               | Yes                  |
| this keyword | Optional         | Required             |
| Syntax       | Simple           | Traditional OOP      |
