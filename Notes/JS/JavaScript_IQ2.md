# 66. What is an object in JavaScript?

A JavaScript object is a **collection of key-value pairs**, where:

* Key = property name
* Value = property value (can be anything: string, number, array, function, another object)

Objects are used to store related data and behavior together.

Think of an object as a real-world entity.

Example: A car has:

* brand
* model
* color
* speed

In JavaScript:

```javascript
const car = {
    brand: "BMW",
    model: "X5",
    color: "Black",
    speed: 180
};
```

Here:

* `brand` → key
* `"BMW"` → value

---

## Why Objects are Important?

Objects help us:

* Organize data
* Represent real-world entities
* Store methods/functions
* Build scalable applications

Example:

```javascript
const student = {
    name: "Rishi",
    age: 24,
    course: "MCA"
};
```

Instead of:

```javascript
let name = "Rishi";
let age = 24;
let course = "MCA";
```

Using objects is cleaner.

---

## Objects Can Store Different Data Types

```javascript
const user = {
    name: "Rishi",
    age: 24,
    isActive: true,
    skills: ["Java", "React", "Node"],
    address: {
        city: "Indore"
    }
};
```

Object values can be:

* String
* Number
* Boolean
* Array
* Function
* Another object

---

## Objects Can Also Have Methods

Methods are functions inside objects.

```javascript
const person = {
    name: "Rishi",
    greet: function() {
        console.log("Hello");
    }
};

person.greet();
```

Output:

```javascript
Hello
```

---

## Accessing Object Properties

### Dot notation

```javascript
console.log(person.name);
```

### Bracket notation

```javascript
console.log(person["name"]);
```

---

## Object Memory Behavior

Objects are **reference types**.

Example:

```javascript
const obj1 = { name: "Rishi" };
const obj2 = obj1;

obj2.name = "Singh";

console.log(obj1.name);
```

Output:

```javascript
Singh
```

Because both point to same memory.

---

## Interview Answer (Short)

**An object in JavaScript is a non-primitive data type used to store collections of key-value pairs. It helps organize related data and functions together. Objects are highly flexible and are used extensively in JavaScript applications.**

# 67. How do you create objects?

There are multiple ways to create objects in JavaScript.

Main methods:

1. Object Literal
2. new Object()
3. Constructor Function
4. Class
5. Object.create()

---

# 1. Using Object Literal (Most Common)

```javascript
const user = {
    name: "Rishi",
    age: 24
};
```

Simple and most used.

---

# 2. Using new Object()

```javascript
const user = new Object();

user.name = "Rishi";
user.age = 24;
```

Same result, less preferred.

---

# 3. Using Constructor Function

```javascript
function User(name, age) {
    this.name = name;
    this.age = age;
}

const user1 = new User("Rishi", 24);
```

Useful for creating multiple objects.

---

# 4. Using Class

Modern JavaScript uses classes.

```javascript
class User {
    constructor(name, age) {
        this.name = name;
        this.age = age;
    }
}

const user1 = new User("Rishi", 24);
```

---

# 5. Using Object.create()

Creates object from prototype.

```javascript
const person = {
    greet() {
        console.log("Hello");
    }
};

const user = Object.create(person);
user.name = "Rishi";
```

---

## Interview Answer (Short)

**Objects in JavaScript can be created using object literals, the Object constructor, constructor functions, classes, and Object.create(). The most commonly used method is object literal syntax.**

# 69. What is object destructuring?

Object destructuring is a JavaScript feature that allows extracting values from objects and storing them into variables easily.

Without destructuring:

```javascript
const user = {
    name: "Rishi",
    age: 24
};

const name = user.name;
const age = user.age;
```

With destructuring:

```javascript
const user = {
    name: "Rishi",
    age: 24
};

const { name, age } = user;
```

Cleaner and shorter.

---

## Example

```javascript
const person = {
    firstName: "Rishi",
    city: "Indore"
};

const { firstName, city } = person;

console.log(firstName);
console.log(city);
```

Output:

```javascript
Rishi
Indore
```

---

## Renaming Variables

```javascript
    name: "Rishi"
};

const { name: userName } = user;

console.log(userName);
```

Output:

```javascript
Rishi
```

# 71. What is shallow copy?

A shallow copy means creating a new object, but only copying the **top-level properties**.

If the object contains nested objects or arrays, those nested structures are **not copied separately**. Instead, their references are copied.

That means:

* Primitive values → copied by value
* Nested objects/arrays → copied by reference

---

## Example

```javascript
const user1 = {
    name: "Rishi",
    age: 24
};

const user2 = { ...user1 };

console.log(user2);
```

Output:

```javascript
{
   name: "Rishi",
   age: 24
}
```

This is a shallow copy.

---

# Primitive Properties in Shallow Copy

```javascript
const user1 = {
    name: "Rishi"
};

const user2 = { ...user1 };

user2.name = "Singh";

console.log(user1.name);
console.log(user2.name);
```

Output:

```javascript
Rishi
Singh
```

Primitive values are copied independently.

---

# Problem with Nested Objects

```javascript
const user1 = {
    name: "Rishi",
    address: {
        city: "Indore"
    }
};

const user2 = { ...user1 };

user2.address.city = "Bhopal";

console.log(user1.address.city);
```

Output:

```javascript
Bhopal
```

Why?

Because nested object `address` is shared.

Memory representation:

```javascript
user1.address ---> same memory <--- user2.address
```

---

# Ways to Create Shallow Copy

### 1. Spread Operator

```javascript
const copy = { ...original };
```

---

### 2. Object.assign()

```javascript
const copy = Object.assign({}, original);
```

---

### 3. Array methods

```javascript
const arr2 = [...arr1];
```

or

```javascript
const arr2 = arr1.slice();
```

---

# Advantages

* Fast
* Simple
* Good for flat objects

---

# Disadvantages

* Unsafe for nested objects

---

## Interview Answer (Short)

**A shallow copy creates a new object and copies only top-level properties. Nested objects and arrays are copied by reference, meaning changes in nested structures affect both original and copied objects.**

---

---

# 72. What is deep copy?

Deep copy means creating a completely independent copy of an object, including all nested objects and arrays.

Everything is duplicated.

After deep copy:

* Original object → separate memory
* Copied object → separate memory

Changes in copied object do NOT affect original.

---

## Example

```javascript
const user1 = {
    name: "Rishi",
    address: {
        city: "Indore"
    }
};

const user2 = structuredClone(user1);

user2.address.city = "Bhopal";

console.log(user1.address.city);
console.log(user2.address.city);
```

Output:

```javascript
Indore
Bhopal
```

Now both are independent.

---

# Deep Copy Methods

---

## 1. structuredClone()

Modern and best approach.

```javascript
const copy = structuredClone(original);
```

---

## 2. JSON Method

```javascript
const copy = JSON.parse(JSON.stringify(original));
```

Example:

```javascript
const user2 = JSON.parse(JSON.stringify(user1));
```

---

## Limitations of JSON Method

It cannot properly copy:

* Functions
* Date
* Map
* Set
* undefined

Example:

```javascript
const obj = {
    date: new Date()
};
```

JSON method may break data structure.

---

## 3. Manual Recursive Copy

Advanced approach.

Used in libraries like Lodash.

Example:

```javascript
_.cloneDeep(obj);
```

(Using Lodash)

---

# Advantages

* Safe for nested structures
* Prevents unwanted mutations

---

# Disadvantages

* Slower than shallow copy
* More memory usage

---

## Interview Answer (Short)

**A deep copy creates a fully independent duplicate of an object, including all nested objects and arrays. Changes made to the copied object do not affect the original object.**

# 76. What is prototype?

In JavaScript, every object has a hidden internal property that links to another object called its **prototype**.

Prototype allows objects to inherit properties and methods from other objects.

In simple words:

> Prototype is an object from which another object can inherit properties and methods.

JavaScript uses prototypes to implement inheritance.

---

## Example

```javascript
const person = {
    greet() {
        console.log("Hello");
    }
};

const user = Object.create(person);

user.greet();
```

Output:

```javascript
Hello
```

Why does this work?

Because:

* `user` does not have `greet()`
* JavaScript checks its prototype (`person`)
* Finds `greet()`
* Executes it

---

# Prototype in Constructor Functions

Every function in JavaScript has a special property called `prototype`.

Example:

```javascript
function Person(name) {
    this.name = name;
}
```

Check prototype:

```javascript
console.log(Person.prototype);
```

Output:

```javascript
{ constructor: f }
```

You can add methods to it:

```javascript
Person.prototype.sayHello = function() {
    console.log("Hello " + this.name);
};
```

Create object:

```javascript
const p1 = new Person("Rishi");

p1.sayHello();
```

Output:

```javascript
Hello Rishi
```

---

## Why Use Prototype?

Without prototype:

```javascript
function User(name) {
    this.name = name;
    this.greet = function() {
        console.log("Hello");
    };
}
```

Every object gets its own copy of `greet()`.

Bad for memory.

Using prototype:

```javascript
function User(name) {
    this.name = name;
}

User.prototype.greet = function() {
    console.log("Hello");
};
```

Now all objects share same method.

Efficient.

---

## Interview Answer (Short)

**A prototype in JavaScript is an object from which other objects inherit properties and methods. It is the foundation of JavaScript’s inheritance system.**

---

# 77. What is prototype chaining?

Prototype chaining is the mechanism by which JavaScript searches for properties or methods through linked prototypes.

When you access a property:

1. JavaScript checks current object
2. If not found → checks prototype
3. If not found → checks prototype’s prototype
4. Continues until `null`

This is called prototype chain.

---

## Example

```javascript
const animal = {
    eats: true
};

const dog = Object.create(animal);

dog.barks = true;
```

Now:

```javascript
console.log(dog.barks);
```

Found directly.

Output:

```javascript
true
```

---

But:

```javascript
console.log(dog.eats);
```

Not in dog.

JavaScript searches prototype (`animal`).

Output:

```javascript
true
```

---

Prototype chain:

```javascript
dog → animal → Object.prototype → null
```

---

## Another Example

```javascript
const arr = [1, 2, 3];
```

Can you use:

```javascript
arr.push(4)
```

Why?

Because:

* `push()` not inside array instance directly
* It exists in `Array.prototype`

Chain:

```javascript
arr → Array.prototype → Object.prototype → null
```

---

## Benefits

* Code reuse
* Memory optimization
* Inheritance support

---

## Interview Answer (Short)

**Prototype chaining is the process by which JavaScript looks for properties or methods in an object and its prototype chain until the property is found or the chain ends at null.**

---

# 82. What is Object.freeze()?

`Object.freeze()` prevents modifications to an object.

After freezing:

* Cannot add properties
* Cannot delete properties
* Cannot modify properties

Object becomes immutable (at top level).

Syntax:

```javascript
Object.freeze(object)
```

---

## Example

```javascript

    name: "Rishi",
    age: 24
};

Object.freeze(user);

user.name = "Singh";
user.city = "Indore";
delete user.age;

console.log(user);
```

Output:

```javascript
{
   name: "Rishi",
   age: 24
}
```

Nothing changes.

---

# What Freeze Prevents

| Operation       | Allowed? |
| --------------- | -------- |
| Modify property | ❌        |
| Add property    | ❌        |
| Delete property | ❌        |

---

Example:

```javascript

    name: "Rishi",
    address: {
        city: "Indore"
    }
};

Object.freeze(user);

user.address.city = "Bhopal";

console.log(user.address.city);
```

Output:

```javascript
Bhopal
```

Nested objects are still mutable.

---

## Deep Freeze

Requires recursive freezing.

---

## Use Cases

* Protect configuration objects
* Immutable data handling
* Prevent accidental changes

---

## Interview Answer (Short)

**Object.freeze() makes an object immutable by preventing addition, deletion, or modification of properties. However, it only performs shallow freezing.**

# Double Equals (`==`)

Loose equality operator.

It compares values **after type conversion (type coercion)**.

Example:

```javascript id="9p8vw6"

```

Output:

```javascript id="6o5q2m"
true
```

Why?

Because JavaScript converts `"5"` → `5`

Then compares.

---

Example:

```javascript id="1t7qzw"

```

Output:

```javascript id="q3hv9l"
true
```

Because JS converts:

* false → 0

---

More weird examples:

```javascript id="wgbfjg"
console.log("" == 0);
```

Output:

```javascript id="om13vl"
true
```

---

```javascript id="m62b84"
console.log(null == undefined);
```

Output:

```javascript id="zsp4xr"
true
```

Because of coercion rules.

---

# Triple Equals (`===`)

Strict equality operator.

Checks:

* Value
* Type

No type conversion.

Example:

```javascript id="2hn4q9"
console.log(5 === "5");
```

Output:

```javascript id="mxydsj"
false
```

Because:

* number ≠ string

---

Example:

```javascript id="x39w7z"
console.log(false === 0);
```

Output:

```javascript id="lkdd2v"
false
```

---

# Comparison Table

| Operator | Type Conversion | Checks       |
| -------- | --------------- | ------------ |
| ==       | Yes             | Value only   |
| ===      | No              | Value + Type |

---

# Best Practice

Always prefer:

```javascript id="vk8rz7"
===
```

Because it avoids unexpected behavior.

---

## Interview Answer (Short)

**`==` compares values after performing type coercion, while `===` compares both value and type without type conversion. In modern JavaScript, `===` is preferred for safer comparisons.**

# 111. What is an Array?

An **Array** in JavaScript is a special object used to store **multiple values in a single variable**. These values can be of any data type such as numbers, strings, booleans, objects, functions, or even other arrays.

Instead of creating separate variables for each value, arrays help organize data in an ordered collection.

## Example:

```javascript
let fruits = ["Apple", "Banana", "Mango"];
```

Here:

* `fruits[0] = "Apple"`
* `fruits[1] = "Banana"`
* `fruits[2] = "Mango"`

---

## Key Features of Arrays

### 1. Ordered Collection

Elements are stored in a fixed order.

```javascript
let arr = [10, 20, 30];
```

---

### 2. Index-Based

Array indexing starts from **0**.

```javascript
console.log(arr[0]); // 10
console.log(arr[2]); // 30
```

---

### 3. Dynamic Size

JavaScript arrays are dynamic, meaning size can change.

```javascript
let nums = [1, 2];
nums.push(3);
console.log(nums); // [1, 2, 3]
```

---

### 4. Can Store Mixed Data Types

Unlike many languages, JavaScript arrays can contain mixed types.

```javascript
let mixed = [10, "Hello", true, {name: "Rishi"}];
```

---

## Array Internally

Even though arrays look special, internally they are objects.

```javascript
let arr = [1, 2, 3];

console.log(typeof arr); // object
```

That’s why arrays have built-in methods like:

* push()
* pop()
* map()
* filter()
* reduce()

---

## Common Array Methods

* `push()` → Add element at end
* `pop()` → Remove last element
* `shift()` → Remove first element
* `unshift()` → Add at beginning

Example:

```javascript
let numbers = [1, 2, 3];

numbers.push(4);
console.log(numbers); // [1,2,3,4]

numbers.pop();
console.log(numbers); // [1,2,3]
```

---

## Real-Life Example

Suppose you are building an e-commerce website.


All cart items can be stored inside an array.

# 116. What is reduce()?

The `reduce()` method in JavaScript is used to **reduce an array into a single value** by applying a callback function on each element.

That single value can be:

* Sum
* Product
* Object
* Array
* String
* Any accumulated result

It is one of the most powerful array methods.

---

## Syntax

```javascript
array.reduce((accumulator, currentValue, index, array) => {
    return updatedAccumulator;
}, initialValue);
```

---

## Parameters

* **accumulator** → Stores accumulated result
* **currentValue** → Current array element
* **initialValue** → Starting value of accumulator

---

## How reduce Works

Suppose:

```javascript
let nums = [1, 2, 3, 4];
```

We want sum.

Step-by-step:

* 0 + 1 = 1
* 1 + 2 = 3
* 3 + 3 = 6
* 6 + 4 = 10

Final result = 10

---

## Example 1: Sum

```javascript
let nums = [1, 2, 3, 4];

let sum = nums.reduce((acc, curr) => acc + curr, 0);

console.log(sum);
```

Output:

```javascript
10
```

---

## Example 2: Product

```javascript
let nums = [2, 3, 4];

let product = nums.reduce((acc, curr) => acc * curr, 1);

console.log(product);
```

Output:

```javascript
24
```

---

## Example 3: Find Maximum

```javascript
let nums = [5, 10, 25, 8];

let max = nums.reduce((acc, curr) => {
    return curr > acc ? curr : acc;
});

console.log(max);
```

Output:

```javascript
25
```

---

## Real Use Case: Total Cart Price

```javascript

    { product: "Keyboard", price: 2000 }

console.log(total);
```

```javascript
53000
```

## Why Use reduce?

Useful for:

* Sum calculations
* Grouping data
* Counting occurrences
* Flattening arrays
* Building objects

---

## Interview Answer

**“reduce() is a higher-order array method that processes all elements of an array and reduces them into a single value. It uses an accumulator to store intermediate results and is commonly used for sum, product, aggregation, and complex data transformations.”**

# 135. How do you remove duplicates from an array?

There are multiple ways to remove duplicates in JavaScript.

Most common interview question.

---

# Method 1: Using Set (Best)

A Set stores only unique values.

```javascript id="vhoq2h"
let nums = [1, 2, 2, 3, 4, 4];

let unique = [...new Set(nums)];

console.log(unique);
```

Output:

```javascript id="0y7hy6"
[1, 2, 3, 4]
```

---

Why best?

* Clean
* Fast
* Modern approach

---

# Method 2: Using filter()

```javascript id="t81ftv"
let nums = [1, 2, 2, 3];

let unique = nums.filter((value, index, array) => {
    return array.indexOf(value) === index;
});

console.log(unique);
```

Output:

```javascript id="n5g9v2"
[1, 2, 3]
```

---

# Method 3: Using Loop

```javascript id="0m0r1j"
let nums = [1, 2, 2, 3];
let unique = [];

for (let num of nums) {
    if (!unique.includes(num)) {
        unique.push(num);
    }
}
```

# Removing Duplicate Objects

Example:

```javascript id="b2t1pi"
let users = [
    { id: 1, name: "Rishi" },
    { id: 1, name: "Rishi" }
];
```

Using Map:

```javascript id="3v7r14"
let unique = [...new Map(users.map(user => [user.id, user])).values()];
```
