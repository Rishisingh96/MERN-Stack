6. Difference Between <script>, <script defer> and <script async>
This is a very important interview question related to page performance and loading behavior.
________________________________________
1. <script> (Normal Script)
Behavior
•	HTML parsing stops
•	Script is downloaded & executed
•	HTML parsing resumes
Impact
•	Slows page loading
Example
<script src="app.js"></script>
Use Case
•	Small scripts
•	When script depends on HTML above it
________________________________________
2. <script defer>
Behavior
•	Script downloads in parallel
•	Execution happens after HTML parsing
•	Maintains script order
Impact
•	Better performance
•	Safe DOM access
Example
<script src="app.js" defer></script>
Best Use Case
•	DOM-dependent scripts
•	Large JS files
________________________________________
3. <script async>
Behavior
•	Script downloads in parallel
•	Executes immediately when downloaded
•	Does not wait for HTML parsing
•	Execution order is not guaranteed
Example
<script src="app.js" async></script>
Best Use Case
•	Independent scripts
•	Analytics, ads
________________________________________
Comparison Table (Must Remember)
Feature	<script>	<script defer>	<script async>
HTML Parsing	Blocked	Not blocked	Not blocked
Script Execution	Immediately	After HTML load	Immediately
Execution Order	Maintained	Maintained	Not guaranteed
DOM Access	Limited	Safe	Unsafe
Performance	Slow	Best	Fast but risky
________________________________________
Execution Timeline (Interview Explanation)
Normal Script:
HTML ----[STOP]---- JS ---- HTML

Defer:
HTML ----------- JS (after HTML)

Async:
HTML ---- JS (anytime)

Final Interview-Ready Summary
Client-side JavaScript runs in the browser and handles UI logic, while server-side JavaScript runs on the server and manages business logic and data.
The <script>, <script defer>, and <script async> tags control how and when JavaScript loads and executes, directly impacting web performance.

Definition
JavaScript written directly inside HTML elements.

Advantages
•	Simple for small actions
•	Easy to write
Disadvantages
•	Poor readability
•	Mixing HTML and JS
•	Not reusable
________________________________________
2. Internal JavaScript
Definition
JavaScript written inside <script> tag within the same HTML file.
Example
<script>
  function showMsg() {
    alert("Hello World");
  }
</script>
Advantages
•	Better than inline
•	Suitable for small projects
Disadvantages
•	Not reusable across pages
•	Larger HTML files
________________________________________
3. External JavaScript
Definition
JavaScript written in a separate .js file and linked to HTML.
Example
<script src="app.js"></script>
Advantages
•	Clean separation of concerns

9. Difference between var, let, and const in JavaScript
In JavaScript, var, let, and const are used to declare variables, but they differ in scope, hoisting behavior, reassignability, redeclaration rules, and best use cases.
________________________________________
1️. var (Old JavaScript – Function Scoped)
var was introduced in ES5 and is the oldest way to declare variables.
🔹 Key Characteristics:
•	Function-scoped (NOT block-scoped)
•	Hoisted and initialized with undefined
•	Allows redeclaration
•	Allows reassignment
•	Can cause unexpected bugs
🔹 Example:

  if (true) {
    var x = 10;
  }
  console.log(x); // 10 (accessible outside block)
}

var a = 5;
👉 Variable is hoisted but value is assigned later.
❌ Problems with var:
•	No block scope
•	Can be accessed before declaration
•	Leads to scope leakage

let was introduced in ES6 (2015) to fix issues with var.
🔹 Key Characteristics:
•	Block-scoped ({})
•	Hoisted but not initialized
•	Cannot be redeclared in same scope
•	Can be reassigned
•	Exists in Temporal Dead Zone (TDZ)
🔹 Example:
if (true) {
  let y = 20;
}
console.log(y); // ReferenceError
🔹 Temporal Dead Zone:
console.log(b); // ReferenceError
let b = 10;
👉 Variable exists but cannot be used before declaration.
________________________________________

const is used to declare variables whose reference cannot change.
🔹 Key Characteristics:
•	Block-scoped
•	Must be initialized at declaration
•	Cannot be redeclared
•	Cannot be reassigned
•	Object/array values can be modified
🔹 Example:

PI = 3.15; // TypeError
🔹 Objects with const:
const user = { name: "Rishi" };
user.name = "Singh"; // Allowed
________________________________________

Must Initialize	❌	❌	✔
Introduced In	ES5	ES6	ES6

Regular Function

In JavaScript, parameters and arguments are closely related but serve different roles in functions.

Arguments are the actual values passed to a function when it is called.

In the function call.
•	Example:
greet("Rishi", 24); // "Rishi" and 24 are arguments
•	Key Points:
o	Supplied by the caller of the function.
o	Can be more or fewer than parameters (extra arguments are ignored or accessed via arguments object or rest parameters).

Represents	Placeholder / expected input	Actual value passed

39. What Are Callback Functions?
________________________________________
Definition
A callback function is a function passed as an argument to another function, which is then invoked inside the outer function after a certain task or event.
•	Purpose: Allows asynchronous or deferred execution.
•	Core Idea: “Do something after this task is done.”
________________________________________
1️⃣ Basic Example
function greet(name, callback) {
    console.log(`Hello ${name}`);
    callback();
}

function sayGoodbye() {
    console.log("Goodbye!");
}

greet("Rishi", sayGoodbye);
Output:
Hello Rishi
Goodbye!
•	sayGoodbye is a callback executed after greet runs.
________________________________________
2️⃣ Why Callbacks Are Important
1️⃣ Asynchronous Programming
•	Callbacks are essential for tasks that take time like:
o	Network requests (AJAX, fetch)
o	Timers (setTimeout, setInterval)
o	File reading
setTimeout(() => {
    console.log("Executed after 2 seconds");
}, 2000);
2️⃣ Event Handling
•	DOM events often use callbacks:
document.getElementById("btn").addEventListener("click", () => {
    alert("Button clicked!");
});
3️⃣ Higher-Order Functions
•	Functions like map, filter, reduce use callbacks:
const nums = [1, 2, 3];
const squared = nums.map(x => x * x); // x => x*x is a callback
console.log(squared); // [1, 4, 9]

3️⃣ Synchronous vs Asynchronous Callbacks
Type	Example	Notes
Synchronous	[1,2,3].forEach(x => console.log(x))	Runs immediately in sequence

•	Callback Hell / Pyramid of Doom
doTask1(() => {
    doTask2(() => {
        doTask3(() => {
            console.log("Done!");
        });
    });
});
•	Can be solved with Promises or async/await.

🔥 40. Explain Higher-Order Functions with Example
________________________________________
Definition
A higher-order function (HOF) is a function that either:
1.	Takes one or more functions as arguments, or
2.	Returns a function as its result
In short: Functions that operate on other functions.
This concept is a core part of functional programming in JavaScript.
________________________________________
1️⃣ Higher-Order Function Examples
Example 1: Function Taking Another Function as Argument (Callback)
function greet(name, formatter) {
    return "Hello " + formatter(name);
}

function upperCaseName(name) {
    return name.toUpperCase();
}

console.log(greet("Rishi", upperCaseName)); // Hello RISHI
•	greet is a higher-order function because it accepts another function (upperCaseName) as an argument.
•	upperCaseName is called inside greet, allowing dynamic behavior.
________________________________________
Example 2: Function Returning Another Function
function multiplyBy(factor) {
    return function (num) {
        return num * factor;
    };
}

const double = multiplyBy(2);
const triple = multiplyBy(3);

console.log(double(5)); // 10
console.log(triple(5)); // 15
•	multiplyBy is a higher-order function because it returns a function.
•	This enables function factories and closures.
________________________________________
2️⃣ Common JavaScript Higher-Order Functions
JavaScript has many built-in higher-order functions, especially for arrays:
Function	Type	Example
map()	Accepts callback	[1,2,3].map(x => x*2) → [2,4,6]
filter()	Accepts callback	[1,2,3].filter(x => x>1) → [2,3]
reduce()	Accepts callback	[1,2,3].reduce((acc,x)=>acc+x,0) → 6
sort()	Accepts callback	[3,1,2].sort((a,b)=>a-b) → [1,2,3]
________________________________________
3️⃣ Why Use Higher-Order Functions?
1.	Code Reusability
o	Define a function once and apply different callbacks.
2.	Abstraction
o	Focus on what you want to do, not how it is done internally.
3.	Functional Programming
o	Enables pure functions, immutability, and chaining.
4.	Dynamic Behavior
o	Functions can be customized by passing different functions.
________________________________________
4️⃣ Advanced Example: Chaining Higher-Order Functions
const numbers = [1, 2, 3, 4, 5];

const result = numbers
    .filter(n => n % 2 === 0) // keep even numbers
    .map(n => n * 10)         // multiply each by 10
    .reduce((sum, n) => sum + n, 0); // sum them

console.log(result); // 60
•	filter, map, and reduce are all higher-order functions.
•	Enables concise, readable, and functional code.
________________________________________

•	HOF accepts functions OR returns functions.
•	Promotes modularity, abstraction, and functional programming.
•	Widely used in callbacks, event handling, array methods, and functional programming libraries (like Lodash or RxJS).
