# 1. What is React?

### Answer:

**React** is a popular open-source JavaScript library used for building fast, interactive, and reusable User Interfaces (UI), especially for Single Page Applications (SPAs). It was developed by Jordan Walke and is maintained by [Meta](https://www.meta.com?utm_source=chatgpt.com).

React follows a **component-based architecture**, which means the UI is divided into small independent pieces called **components**. Each component manages its own logic and rendering, making applications easier to develop, maintain, and scale.

Unlike traditional web development where the entire page reloads after every interaction, React updates only the necessary parts of the webpage using a concept called the **Virtual DOM**, resulting in faster performance and a better user experience.

### Example:

Instead of writing the same Navbar code on multiple pages:

```jsx
function Navbar() {
  return (
    <nav>
      <h1>My Website</h1>
    </nav>
  );
}
```

You can create the Navbar component once and reuse it throughout the application.

### Key Characteristics of React:

* Component-Based Architecture
* Virtual DOM for fast rendering
* Reusable UI Components
* One-Way Data Flow
* Declarative Programming
* Strong Community Support
* Rich Ecosystem

# 2. Why Was React Created?

### Answer:

Before React was introduced, developers mainly used traditional JavaScript libraries and direct DOM manipulation to update web pages.

As web applications became more complex, managing UI updates became difficult because:

* Code became harder to maintain.
* DOM manipulation was slow.
* Reusable UI structures were limited.
* Large applications became difficult to scale.
* UI and data synchronization became challenging.

To solve these problems, React was created by Facebook in 2011 and publicly released in 2013.

### Problems React Solved:

#### 1. Complex UI Management

Large applications contain many UI elements.

Example:

* Navigation Bar
* Sidebar
* Dashboard
* User Profile
* Notifications

Managing all these manually becomes difficult.

React introduced components to organize UI efficiently.

---

#### 2. Performance Issues with DOM Manipulation

Traditional JavaScript directly updates the DOM.

Example:

```javascript
document.getElementById("title").innerHTML = "Hello";
```

Direct DOM operations are expensive because the browser must re-render elements repeatedly.

React introduced the **Virtual DOM**.

Process:

1. React creates a Virtual DOM copy.
2. Compares old and new versions.
3. Finds differences.
4. Updates only changed elements.

Result:

* Faster rendering
* Better performance
* Improved user experience

---

#### 3. Code Reusability

Before React:

Developers often duplicated HTML and JavaScript code.

With React:

```jsx
<Button />
<Button />
<Button />
```

A single component can be reused multiple times.

Benefits:

* Less code
* Easier maintenance
* Better scalability

---

#### 4. Better User Experience

Users expect:

* Instant updates
* Smooth interactions
* Fast page transitions

React enables:

* Dynamic content updates
* Real-time UI changes
* Faster page interactions

without refreshing the entire page.

### Interview Conclusion:

> React was created to solve problems related to complex UI management, inefficient DOM manipulation, code duplication, and poor scalability. It introduced component-based architecture and Virtual DOM, making web applications faster, maintainable, and easier to develop.

# 5. What is SPA (Single Page Application)?

### Answer:

A **Single Page Application (SPA)** is a web application that loads a single HTML page initially and dynamically updates content without reloading the entire page.

In traditional websites, every user action requests a new page from the server.

### Traditional Website Flow

```text
User Clicks Link
       ↓
Browser Sends Request
       ↓
Server Generates New Page
       ↓
Entire Page Reloads
```

This causes:

* Slower navigation
* Increased server load
* Poor user experience

---

### SPA Flow

```text
User Clicks Link
       ↓
JavaScript Handles Request
       ↓
Only Required Data Fetched
       ↓
UI Updated Dynamically
```

No full page reload occurs.

---

## Example

### Traditional Website

```text
Home → Reload
About → Reload
Contact → Reload
```

### SPA

```text
Home → Instant
About → Instant
Contact → Instant
```

# 6. What is Virtual DOM?

### Answer:

**Virtual DOM (VDOM)** is a lightweight, in-memory representation (copy) of the Real DOM maintained by React. It is a programming concept where React keeps a virtual representation of the UI and synchronizes it with the Real DOM efficiently.

The main purpose of the Virtual DOM is to improve application performance by minimizing expensive direct manipulations of the Real DOM.

### Understanding the Need for Virtual DOM

In traditional JavaScript applications, whenever data changes, the browser may need to:

1. Update the DOM.
2. Recalculate styles.
3. Recalculate layout.
4. Repaint the screen.

These operations are expensive because the Real DOM is directly connected to the browser's rendering engine.

Example:

```javascript
document.getElementById("count").innerText = count;
```

If an application contains thousands of elements, frequent updates can significantly reduce performance.

React solves this problem using the Virtual DOM.

---

## What Exactly is Virtual DOM?

Think of Virtual DOM as:

```text
Real DOM
   ↓
Virtual Copy (Virtual DOM)
```

React first updates the Virtual DOM instead of directly updating the Real DOM.

The Virtual DOM exists only in memory and is much faster to manipulate than the Real DOM.

---

## Example

Suppose the UI is:

```html
<h1>Count: 0</h1>
```

When count changes to 1:

```html
<h1>Count: 1</h1>
```

React:

1. Creates a new Virtual DOM.
2. Compares it with the previous Virtual DOM.
3. Finds the changed part.
4. Updates only the affected node in the Real DOM.

Instead of re-rendering the entire page, React updates only:

```html
<h1>Count: 1</h1>
```

---

## Benefits of Virtual DOM

### 1. Faster Rendering

Only modified elements are updated.

### 2. Better Performance

Reduces expensive DOM operations.

### 3. Efficient UI Updates

React updates only necessary parts.

### 4. Improved User Experience

Applications feel smoother and more responsive.

### 5. Simplified Development

Developers focus on state changes rather than manual DOM manipulation.

---

## Interview Conclusion

> Virtual DOM is a lightweight JavaScript representation of the Real DOM. React uses it to compare UI changes and update only the modified elements in the Real DOM, resulting in faster rendering and improved performance.

---

# 7. How Does Virtual DOM Work?

### Answer:

Virtual DOM works through a process called **Reconciliation** and **Diffing Algorithm**.

Whenever data changes, React follows a series of steps to efficiently update the UI.

---

## Step-by-Step Process

### Step 1: Initial Render

React creates:

```text
Component
    ↓
Virtual DOM
    ↓
Real DOM
```

Example:

```jsx
function App() {
  return <h1>Hello</h1>;
}
```

React converts JSX into Virtual DOM objects and then renders them in the Real DOM.

---

### Step 2: State Changes

Suppose:

```jsx
const [count, setCount] = useState(0);
```

User clicks:

```jsx
setCount(1);
```

React does not immediately update the Real DOM.

Instead:

```text
State Change
      ↓
New Virtual DOM Created
```

---

### Step 3: Diffing

React compares:

```text
Old Virtual DOM
       VS
New Virtual DOM
```

Example:

Old:

```html
<h1>Count: 0</h1>
```

New:

```html
<h1>Count: 1</h1>
```

React detects:

```text
Only text changed
```

---

### Step 4: Reconciliation

React determines the minimum number of changes needed.

Instead of:

```text
Delete Entire Element
Create New Element
```

React performs:

```text
Update Text Only
```

---

### Step 5: Update Real DOM

React updates only the changed node.

```text
Old DOM
<h1>Count: 0</h1>

New DOM
<h1>Count: 1</h1>
```

This process is extremely fast.

---

## Complete Flow

```text
State Change
      ↓
New Virtual DOM Created
      ↓
Diffing Algorithm
      ↓
Find Differences
      ↓
Reconciliation
      ↓
Update Real DOM
```

---

## Interview Conclusion

> React's Virtual DOM works by creating a new virtual representation whenever state changes occur. React compares the old and new Virtual DOM using the Diffing Algorithm, identifies the differences, and updates only the changed parts of the Real DOM through Reconciliation.

---

# 8. What is the Difference Between DOM and Virtual DOM?

### Answer:

The DOM and Virtual DOM are both representations of a webpage, but they differ significantly in performance and implementation.

| Feature      | DOM                                         | Virtual DOM                  |
| ------------ | ------------------------------------------- | ---------------------------- |
| Definition   | Actual representation of webpage in browser | Lightweight copy of Real DOM |
| Location     | Browser Memory                              | JavaScript Memory            |
| Update Speed | Slower                                      | Faster                       |
| Manipulation | Direct                                      | Indirect                     |
| Performance  | Expensive                                   | Efficient                    |
| Re-rendering | Entire affected section                     | Only changed elements        |
| Managed By   | Browser                                     | React                        |
| Memory Usage | Higher                                      | Lower                        |
| Optimization | Limited                                     | Highly Optimized             |

---

## Real DOM Example

```javascript
document.getElementById("title").innerHTML = "Hello";
```

Every change affects the browser directly.

---

## Virtual DOM Example

```jsx
setTitle("Hello");
```

React:

```text
State Change
      ↓
Virtual DOM Update
      ↓
Compare Changes
      ↓
Real DOM Update
```

---

## Real DOM Process

```text
Data Change
      ↓
Real DOM Update
      ↓
Layout Recalculation
      ↓
Repaint
```

---

## Virtual DOM Process

```text
Data Change
      ↓
Virtual DOM Update
      ↓
Diffing
      ↓
Minimal Real DOM Update
```

---

## Why Virtual DOM is Faster?

Because:

* DOM operations are expensive.
* Virtual DOM operations occur in memory.
* React updates only necessary nodes.

---

## Interview Conclusion

> The Real DOM is the actual browser representation of a webpage, while the Virtual DOM is a lightweight JavaScript copy maintained by React. Virtual DOM improves performance by comparing changes and updating only the necessary parts of the Real DOM.

# 9. What is JSX?

### Answer:

**JSX (JavaScript XML)** is a syntax extension for JavaScript that allows developers to write HTML-like code inside JavaScript.

JSX makes React code more readable, maintainable, and easier to understand.

Example:

```jsx
const element = <h1>Hello React</h1>;
```

Although it looks like HTML, it is actually JavaScript.

React converts JSX into JavaScript using a transpiler such as Babel.

---

## Without JSX

```javascript
const element = React.createElement(
  "h1",
  null,
  "Hello React"
);
```

---

## With JSX

```jsx
const element = <h1>Hello React</h1>;
```

JSX is easier to read and write.

---

## How JSX Works Internally

JSX:

```jsx
const element = <h1>Hello React</h1>;
```

Babel converts it into:

```javascript
const element = React.createElement(
  "h1",
  {},
  "Hello React"
);
```

React then creates Virtual DOM objects.

---

## JSX Supports JavaScript Expressions

Example:

```jsx
const name = "Rishi";

return <h1>Hello {name}</h1>;
```

Output:

```html
Hello Rishi
```

---

## JSX Supports Conditions

```jsx
{
  isLoggedIn ? <Dashboard /> : <Login />;
}
```

---

## JSX Supports Loops

```jsx
{
  users.map((user) => <li>{user}</li>);
}
```

---

## Advantages of JSX

# 13. What is the Difference Between Functional and Class Components?

### Answer:

React provides two ways to create components:

1. Functional Components
2. Class Components

Today, Functional Components are the recommended approach because of Hooks.

---

# Functional Components

A Functional Component is simply a JavaScript function that returns JSX.

Example:

```jsx
function Welcome() {
  return <h1>Hello React</h1>;
}
```

Or:

```jsx
const Welcome = () => {
  return <h1>Hello React</h1>;
};
```

---

# Class Components

A Class Component is an ES6 class that extends React.Component.

Example:

```jsx
class Welcome extends React.Component {
  render() {
    return <h1>Hello React</h1>;
  }
}
```

---

# Detailed Comparison

| Feature           | Functional Component | Class Component   |
| ----------------- | -------------------- | ----------------- |
| Syntax            | Function             | Class             |
| Easier to Learn   | Yes                  | No                |
| Boilerplate Code  | Less                 | More              |
| Hooks Support     | Yes                  | No                |
| State Management  | useState             | this.state        |
| Lifecycle Methods | useEffect            | componentDidMount |
| Performance       | Better               | Slightly Heavier  |
| Modern React      | Preferred            | Rarely Used       |
| Readability       | High                 | Moderate          |

---

# State Example

### Functional Component

```jsx
import { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <>
      <h1>{count}</h1>
      <button onClick={() => setCount(count + 1)}>
        Increment
      </button>
    </>
  );
}
```

---

### Class Component

```jsx
class Counter extends React.Component {
  state = {
    count: 0
  };

  increment = () => {
    this.setState({
      count: this.state.count + 1
    });
  };

  render() {
    return (
      <>
        <h1>{this.state.count}</h1>
        <button onClick={this.increment}>
          Increment
        </button>
      </>
    );
  }
}
```

---

# Why Functional Components are Preferred Today?

React introduced Hooks in React 16.8.

Hooks allow:

* State Management
* Lifecycle Features
* Context Access
* Side Effects

inside functional components.

This made class components largely unnecessary.

---

## Interview Conclusion

> Functional Components are JavaScript functions that return JSX, while Class Components are ES6 classes that extend React.Component. Modern React development primarily uses Functional Components because they are simpler, more readable, and support Hooks for state and lifecycle management.

# 16. What is the Purpose of `createRoot()`?

### Answer:

`createRoot()` is a method introduced in **React 18** that creates a React root and enables React's modern rendering engine. It is used to tell React where the application should be rendered in the browser DOM.

Before React 18, developers used:

```jsx
ReactDOM.render(<App />, document.getElementById("root"));
```

In React 18, this was replaced by:

```jsx
import ReactDOM from "react-dom/client";
import App from "./App";

const root = ReactDOM.createRoot(
  document.getElementById("root")
);

root.render(<App />);
```

---

## Why Was createRoot() Introduced?

React 18 introduced several advanced features:

* Concurrent Rendering
* Automatic Batching
* Suspense Improvements
* Better Performance
* Faster UI Updates

These features require the new root API.

---

## How createRoot() Works

Suppose your HTML file contains:

```html
<body>
  <div id="root"></div>
</body>
```

React executes:

```jsx
const root = ReactDOM.createRoot(
  document.getElementById("root")
);
```

React now controls everything inside:

```html
<div id="root"></div>
```

Then:

```jsx
root.render(<App />);
```

renders the App component.

---

## Rendering Flow

```text
HTML Root Element
       ↓
createRoot()
       ↓
React Root Created
       ↓
render(<App />)
       ↓
Component Tree Rendered
```

---

## Benefits of createRoot()

### 1. Enables Concurrent Features

React can prioritize important updates.

### 2. Better Performance

More efficient rendering.

### 3. Future-Proof API

Supports future React features.

### 4. Automatic Batching

Multiple state updates are grouped together.

---

## Example

```jsx
import ReactDOM from "react-dom/client";
import App from "./App";

const root = ReactDOM.createRoot(
  document.getElementById("root")
);

root.render(<App />);
```

---

## Interview Conclusion

> `createRoot()` is a React 18 method used to create a React root and connect the React application to a DOM element. It enables modern React features such as Concurrent Rendering and improved performance.

# 20. Why is React Fast?

### Answer:

React is considered fast because it minimizes expensive DOM operations and efficiently updates only the parts of the UI that have changed.

Its performance comes from several optimizations working together.

---

## 1. Virtual DOM

The biggest reason React is fast is the Virtual DOM.

React creates:

```text
Real DOM
     ↓
Virtual DOM Copy
```

When data changes:

```text
New Virtual DOM
       ↓
Compare with Old Virtual DOM
       ↓
Find Differences
       ↓
Update Only Necessary Nodes
```

This avoids unnecessary DOM updates.

---

## 2. Diffing Algorithm

React compares:

```text
Old Virtual DOM
       VS
New Virtual DOM
```

and identifies exactly what changed.

Example:

Old:

```html
<h1>Count: 0</h1>
```

New:

```html
<h1>Count: 1</h1>
```

React updates only the text node.

---

## 3. Reconciliation

After diffing, React updates only the affected elements.

```text
State Change
      ↓
Diffing
      ↓
Reconciliation
      ↓
Minimal DOM Update
```

---

## 4. Component Reusability

Components allow React to isolate updates.

Example:

```text
App
│
├── Navbar
├── Dashboard
└── Footer
```

If Dashboard changes:

```text
Navbar → No Update
Footer → No Update
Dashboard → Update
```

Only the affected component re-renders.

---

## 5. Automatic Batching

React 18 combines multiple updates into one render.

Example:

```jsx
setCount(count + 1);
setLoading(true);
setName("Rishi");
```

Instead of 3 renders:

```text
Render 1
Render 2
Render 3
```

React performs:

```text
Single Optimized Render
```

---

## 6. Efficient Rendering Strategy

React avoids full-page refreshes.

Only changed portions are updated.

---

## 7. Concurrent Rendering (React 18)

React can prioritize important UI updates.

Example:

```text
Typing Input
      ↓
High Priority
```

```text
Large Data Fetch
      ↓
Low Priority
```

This improves responsiveness.

---

## Complete Performance Flow

```text
State Change
      ↓
Virtual DOM
      ↓
Diffing Algorithm
      ↓
Reconciliation
      ↓
Minimal DOM Updates
      ↓
Fast UI Rendering
```

---

## Interview Conclusion

> React is fast because it uses the Virtual DOM, Diffing Algorithm, Reconciliation process, component-based architecture, automatic batching, and concurrent rendering. These optimizations reduce expensive DOM operations and ensure only the necessary parts of the UI are updated.
